# GUI Smoke Checklist

本文档用于 Beta 发布前的人工 GUI 验证。自动化 smoke 可以覆盖窗口初始化、spinbox 结构、导出构建等基础行为，但真实鼠标点击、授权窗口体验和用户可见文案仍建议人工确认。

自动截图位于工作区 `test/beta_2026-07-10/screenshots/`，包含深色/浅色主窗口与 SpinBox/ComboBox 页面。它们由 Qt offscreen + 项目 palette/style 模拟生成，只证明渲染路径成功；当前沙箱无法读取系统中文字体，截图中文字可能显示为方框，不能作为字体或完整视觉验收证据。

## 激活流程

- 首次启动未激活设备时，应进入激活页，而不是直接进入工作台。
- 激活页应显示机器码和申请码。
- **复制申请码** 按钮可点击，剪贴板中应出现申请码文本。
- 新申请码应以 `LFREQ1.` 开头且保持单行；普通用户无需输入版本、machine ID 或 edition。
- 不导入 license 时，**进入工作台** 按钮应不可用或无法进入正式工作台。
- 导入作者签发的有效 `.lic` 后，应显示授权有效，并允许进入工作台。
- 导入无效、损坏、过期或非本机 `.lic` 时，应显示可理解的错误提示。
- 新增任一现有步骤类型后，应立即选中该步骤并显示对应右侧属性编辑页。
- 文件菜单应显示 `Ctrl+S`、`Ctrl+Shift+S`、`Ctrl+R`、`Ctrl+E`，编辑菜单应显示 `Delete`。
- Command 返回 9009、路径不存在或权限不足时，应在保留原始日志和退出码的同时显示中文解释。

## 编排与秒数控件

- 在工作台新增 **等待** 步骤。
- 等待秒数输入框的上/下按钮视觉位置应与实际点击区域一致。
- 鼠标点击上按钮，秒数应增加。
- 鼠标点击下按钮，秒数应减少。
- 手动输入秒数后保存步骤，步骤值应保持正确。
- 应用、网页、命令步骤中的 **后续延迟(秒)** 控件也应通过同样的点击检查。
- 分别在深色与浅色主题肉眼确认上下箭头清晰，且 hover、pressed、disabled 状态没有低对比度或错色。
- 分别在 100%、125%、150% DPI 确认箭头视觉中心与实际点击区域一致。

## Shell ComboBox

- Shell 类型当前值保持为 `cmd` 或 `powershell`，数据绑定不变化。
- 点击文本区、箭头区和箭头左侧均能展开列表。
- 深色/浅色主题下主体与右侧区域应为连续单一控件，不出现双边框、断裂边框或文本遮挡。
- 下拉列表背景、文字、hover、选中和 disabled 状态应清晰。

## 启动项与试运行

- 可以新增 `.exe` 启动项。
- 可以新增 `.bat` / `.cmd` 启动项。
- 可以新增 `.ps1` 启动项。
- 源码试运行应能执行等待步骤。
- 源码试运行应能执行脚本步骤，并在日志区域显示关键状态。
- 日志抽屉的展开/隐藏按钮应可用。
- 运行 `python --version` 时不得弹出 CMD/Windows Terminal；日志应包含版本文本与退出码 0。
- 运行确定不存在的命令时应显示 stderr、非零退出码和失败状态，不能显示命令成功。
- 运行 `vol C:` 不应出现 `0x800700E8`；随后再次试运行应可正常开始，证明上一 QThread 已完成并释放。

## 编辑器同步与 Application 隔离人工回归（18 项）

1. 新增命令后立即编辑名称和命令，不点“保存此步骤修改”，按 `Ctrl+S` 后重载仍保持最新值。
2. Command 中输入双引号、换行、`%PATH%` 和中文，保存后 JSON 内容不被裁剪或改写。
3. 修改步骤后点击工具栏“保存方案”，确认与 `Ctrl+S` 行为一致。
4. 修改步骤后使用 `Ctrl+Shift+S`，另存文件包含最新草稿。
5. 修改步骤后直接切换到下一步骤，旧步骤名称与参数自动提交。
6. 切换步骤时右侧载入不产生伪 dirty、重复日志或列表闪烁。
7. 修改当前步骤后新增网页/命令/等待，旧步骤先提交，新步骤立即选中。
8. 试运行前修改命令，不点应用按钮，实际执行使用最新命令。
9. 试运行开始后继续编辑 UI，不影响已经创建的运行快照。
10. 导出前修改步骤，导出快照使用最新内容。
11. 保存包含空 Command 的草稿应成功，不强迫用户先补齐。
12. 方案第一步为有效 Application、第二步为空 Command 时，试运行应在任何应用启动前阻断。
13. 上述阻断应自动选中第二步、打开 Command 编辑页并聚焦命令输入框。
14. 空应用路径、空 URL、无效 Shell、无效等待时间分别显示步骤序号、名称和字段提示。
15. 严格预检失败时不得创建 `TrialRunWorker` 或 `BuildWorker`。
16. 从开发 PowerShell 启动 XMind，PowerShell 中不应出现 Electron/Chromium stdout 或 stderr 污染。
17. XMind Application 步骤应立即返回，LaunchFlow 可继续后续编排，不等待 GUI 退出。
18. Command 步骤仍需等待完成并在 LaunchFlow 日志保留 command、stdout、stderr 与 returncode。

## 步骤排序、Tooltip 与反馈入口（20 项）

1. 创建三个不同名称步骤。
2. 将步骤 3 拖到步骤 1 前。
3. 确认插入标记清晰。
4. 确认步骤 ID、类型、参数和延迟未改变。
5. 保存、关闭、重新载入，顺序保持。
6. 试运行按照新顺序执行。
7. 导出 EXE 后按照新顺序执行。
8. 拖动有未 Apply 编辑的步骤，内容不丢失。
9. `Alt+↑` / `Alt+↓` 正确移动步骤。
10. 输入框聚焦时快捷键不误移动。
11. 鼠标停在保存、运行、导出、删除按钮上能看到快捷键。
12. Command tooltip 明确输出进入日志。
13. 正式版试运行仍不弹 CMD、PowerShell 或 Windows Terminal。
14. “反馈问题”对话框可以打开并预览文本。
15. 复制诊断信息成功，且不会自动上传。
16. 打开日志目录成功，开发模式定位到 `LaunchFlow-Dev`。
17. 诊断内容中没有完整 machine ID、request ID、LFREQ1、signature 或 license 内容。
18. 深色主题 tooltip 与拖拽插入标记可读。
19. 浅色主题 tooltip 与拖拽插入标记可读。
20. 100%、125%、150% DPI 下拖拽位置正确。

## Windows 图标

- 源码启动时窗口标题栏应显示 LaunchFlow 图标。
- 发布版启动时窗口、EXE 文件和任务栏应显示 LaunchFlow 图标，而不是通用程序图标。
- 如仍显示旧图标，先关闭所有 LaunchFlow 实例并刷新 Windows 图标缓存后复测；代码侧构建参数已由 release smoke 覆盖。

## 图标、历史载入、拖拽与日志工作区（27 项）

1. `python -m editor.main` 启动后标题栏显示 LaunchFlow 图标。
2. 源码模式任务栏显示 LaunchFlow 图标。
3. 发布版任务栏显示 LaunchFlow 图标。
4. 若显示旧图标，关闭实例并刷新 Windows 图标缓存后复测。
5. 单击历史方案立即载入一次。
6. 方向键移动历史选择不自动载入。
7. Enter 载入当前选中的历史方案。
8. 未保存修改时出现保存、放弃、取消三种选择。
9. 取消后当前方案、草稿和历史选择不丢失。
10. 将第三项移动到第一项无需精确寻找细白线。
11. 拖拽项显示半透明缩放预览。
12. 全宽插入线和左侧箭头清晰。
13. 最终位置提示与松手后的真实位置一致。
14. 顶部与底部扩展投放区有效。
15. 长列表顶部/底部 auto-scroll 正常。
16. 拖拽后保存、重启、试运行和导出顺序一致。
17. 系统、执行、输出、成功、等待/提示和错误容易区分且保留文本标签。
18. 深色主题日志颜色可读。
19. 浅色主题日志颜色可读。
20. 默认日志区域至少约 8 行。
21. 日志工具按钮位于右侧且不占顶部整行。
22. expanded、normal、collapsed 和恢复默认均正常。
23. 日志布局不会把上方编辑区压缩到无法操作。
24. 清空 UI 后磁盘日志仍存在。
25. 复制全部得到纯文本而不是 HTML。
26. 反馈诊断继续掩码路径与敏感标识，不读取 license、不自动上传。
27. 100%、125%、150% DPI 下图标、投放位置和日志布局正常。

## 导出与运行

- 点击 **导出 EXE** 时，如果方案包含本地启动文件，应显示随包携带确认提示。
- 导出的启动器应能运行。
- 导出的启动器执行 `.cmd` / `.ps1` 时，应优先使用包内 `launchflow_assets` 文件。
- 若目标应用依赖外部 DLL、配置目录或数据目录，应确认文档和提示没有承诺自动携带完整依赖树。

## 发布包边界

- 发布包不得包含 `private/private_key.pem`。
- 发布包不得包含 `generated_licenses/*.lic`。
- 发布包不得包含任何真实用户 `.lic` 文件。
- 发布版首次启动后如无 license，应进入激活流程。
- 将 `LaunchFlow.exe` 单独复制到空目录短启动，目录中应仍只有该 EXE；运行数据应出现在测试覆盖目录或 `%LOCALAPPDATA%\LaunchFlow`。
- 备份/卸载人工确认：`plans/`、`licenses/` 可复制备份；删除 EXE 不会自动删除 AppData。
