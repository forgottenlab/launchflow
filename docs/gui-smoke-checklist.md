# GUI Smoke Checklist

本文档用于 Beta 发布前的人工 GUI 验证。自动化 smoke 可以覆盖窗口初始化、spinbox 结构、导出构建等基础行为，但真实鼠标点击、授权窗口体验和用户可见文案仍建议人工确认。

## 激活流程

- 首次启动未激活设备时，应进入激活页，而不是直接进入工作台。
- 激活页应显示机器码和申请码。
- **复制申请码** 按钮可点击，剪贴板中应出现申请码文本。
- 不导入 license 时，**进入工作台** 按钮应不可用或无法进入正式工作台。
- 导入作者签发的有效 `.lic` 后，应显示授权有效，并允许进入工作台。
- 导入无效、损坏、过期或非本机 `.lic` 时，应显示可理解的错误提示。

## 编排与秒数控件

- 在工作台新增 **等待** 步骤。
- 等待秒数输入框的上/下按钮视觉位置应与实际点击区域一致。
- 鼠标点击上按钮，秒数应增加。
- 鼠标点击下按钮，秒数应减少。
- 手动输入秒数后保存步骤，步骤值应保持正确。
- 应用、网页、命令步骤中的 **后续延迟(秒)** 控件也应通过同样的点击检查。

## 启动项与试运行

- 可以新增 `.exe` 启动项。
- 可以新增 `.bat` / `.cmd` 启动项。
- 可以新增 `.ps1` 启动项。
- 源码试运行应能执行等待步骤。
- 源码试运行应能执行脚本步骤，并在日志区域显示关键状态。
- 日志抽屉的展开/隐藏按钮应可用。

## 导出与运行

- 点击 **导出 EXE** 时，如果方案包含本地启动文件，应显示随包携带确认提示。
- 导出的启动器应能运行。
- 导出的启动器执行 `.cmd` / `.ps1` 时，应优先使用包内 `launchflow_assets` 文件。
- 若目标应用依赖外部 DLL、配置目录或数据目录，应确认文档和提示没有承诺自动携带完整依赖树。

## 发布包边界

- 发布包不得包含 `private/private_key.pem`。
- 发布包不得包含 `generated_licenses/*.lic`。
- 发布包不得包含任何真实用户 `.lic` 文件。
- 发布版首次启动后如无 license，应进入激活流程。
